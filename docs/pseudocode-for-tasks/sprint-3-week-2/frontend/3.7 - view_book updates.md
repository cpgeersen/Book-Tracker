# 3.7 - view_book updates

### Main Objectives

Fix some usability issues.

### 3.7.1 - Change Total Chapters Input to only show when user clicks edit (add two buttons and a number input)

#### Requirements:
- Must populate with data for total chapters and completed chapters
  #### Example Initial State:
  ![Mock-up for Chapter Input](https://github.com/cpgeersen/Book-Tracker/blob/main/images/mock-ups/3.7.1_chapter_input.png)
- There is an edit button next to the chapters to edit total chapters with the following functionality:
  - When the user clicks the edit button it makes the chapters editable
  - When the user clicks the edit button it also becomes hidden and displays a new button for saving
    #### Example Initial State:
    ![Mock-up for Total Chapter Edit](https://github.com/cpgeersen/Book-Tracker/blob/main/images/mock-ups/3.7.1_chapter_input_edit_total_chapters.png)
- Clicking the save button sends an UPDATE to the server and saves the new total chapters value and shows the new
value on the frontend and reverts to initial state.


  
### 3.7.2 - Edit functionality for summary (adds two buttons and a text form)
Step 1: User clicks edit button.

Step 2: Edit button is now hidden, a new text input box is shown and a new save button is shown.

Step 3: User can edit summary in the text input box (will have previous summary populate the text input for edits).

Step 4: Once a user is done, clicking the save button will send an UPDATE of the change.

Step 5: The summary reverts back to a text box with the new summary and an edit button.

Note: Just use the user input to update the summary on the frontend, you do not have to requery the database for the new summary


### 3.7.3 - Add save button to tags to commit changes
Step 1: User changes a tag toggle.

Step 2: Save button is now visible next to the tags.

Step 3: User clicks save button, sends an UPDATE to the server with current tag information.

Step 4: Save button no longer visible, until a user toggles one of that tags again.
