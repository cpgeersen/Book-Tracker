# 3.16 - Update View Book Page

### Main Objective:
Improve the usability of the book and note frontend.

### Notes:
Originally the notes were to be delegated to a separate page, but having it on the same page is a good solution. This
will have future implementation of features easier. Given this change though some functional requirements will need
to be changed.

### Requirements:
- Remove the old notes button from the individual book page.
- Move the actions section of the page underneath the book image
- Add a new button in the actions section to upload a cover image
- Move the notes section into a separate accordian section (like the previously implemented secondary author accordian
in the add book form).
- Add a text input for note title.
- Initially hide the new note inputs and save note button until the user click a newly added "add new note" button
  - After successfully adding the new note revert to initial state of hiding the inputs
- Put notes into a scrollable list that displays the title for each entry. When a user clicks one of the notes in the
list, display the note underneath
- When a note is displayed add two buttons (edit and delete)
