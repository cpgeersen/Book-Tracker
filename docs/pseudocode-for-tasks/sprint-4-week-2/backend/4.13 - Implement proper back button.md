# 4.13 - Implement proper back button functionality based on current route

### Main Objectives:
Need to add proper context for the back link in the nav-bar depending on where the user is.

For example, a user searches for a book and clicks on a book result to see an individual book page. When the user
presses the back link in the nav-bar it bring the user to the add book form. Also, we cannot just change the back
link for the individual page to the local search since we want to show the individual book page when a book a created.

Note: I hope I explained it well enough.

### Implementation:
Note: This implementation should work, but I am not fully certain it will.

The proposed implementation is to include a new data endpoint when a `render_template` is called in the main file.
This data will then be used in a flask placeholder on the page to maintain context with the back link.
