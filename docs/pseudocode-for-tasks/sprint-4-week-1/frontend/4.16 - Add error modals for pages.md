# 4.16 - Add error modals for pages using flask extends

### Main Objective
Use HTML `<dialog>` tags to create modals for various errors. Do not have to worry about how they will be used, they
will be very easy to integrate.

### Requirements:
- Use `<dialog>` with the `open` attribute (this will make it open on page load, the way we will be using it)
  - https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/dialog#using_the_dialog_open_attribute
- Must use the flask extends syntax to create modified versions of existing pages
  - https://flask.palletsprojects.com/en/stable/patterns/templateinheritance/
- Must have similar styling to other modals

### Modals Needed:
#### Add Book Page:
- User adds a book already present, inform user
- User tries to add a book with malformed data, inform user
  - Note: This should not be possible unless a user is intentionally trying to
  break the app with HTML form manipulation
#### Local Search Page:
- User enters a nonexistent ISBN in the URL for route `/book/isbn/<isbn>`
  - Should tell the user that this book does not exist
