#### Harder Solution (Most likely will never implement)

Different editions of books can have different ISBNs. This can lead to a user adding the same book twice under
a different ISBN. This can happen two ways:
- User Error:
  - Handled in: Mediator Module
  - Arises when a user adds two books that are the same with two different ISBNs.
- OL-Local Mismatch:
  - Handled in: Mediator Module
  - This occurs when a user adds a book under one ISBN and then searches the same title through the OL interface.
  - It is possible that a returned result from OL gives a different ISBN title match and the user adds the present book
  under a different ISBN
  - Additionally, an OL result that matches an ISBN through a title search needs to be addressed.

Harder Solutions:
- Search Local Database for Title-Author Matches:
  1. First: Query OL normally for data, and get returned result.
  2. Second: We use the OL data to search the local database for any matches on ISBN and add any to the duplicate key in
  JSON result (will explain later).
     - Note: Since a title search can return the book already in the database.
  3. Third: We use OL result to search for Title-Author matches under different ISBN and add any result to the duplicate
  key in JSON result.
  4. Fourth: Give the rest of the OL results that passed the previous tests.
    
    EXAMPLE:
    - User Locally adds 'The Hobbit' under ISBN: 0395520215
    - User Queries through OL: 'Hobbit'
    - OL response: Includes (truncated) {ISBN: 0395520215, title: The Hobbit, author: J.R.R. Tolkien}
                                        {ISBN: 9780547928227, title: The Hobbit, author: J.R.R. Tolkien}
                                        {ISBN: 1857939379, title: The Hobbit Companion, author: David Day}
    - First, we address the duplicate by adding the result to the present key in the result JSON response
      - {present: {ISBN: 0395520215, title: The Hobbit, author: J.R.R. Tolkien}}
    - Second, we look for Title-Author pairs that with different ISBN and then add to duplicate response key
      - {present: [{ISBN: 0395520215, title: The Hobbit, author: J.R.R. Tolkien}],
         duplicate: [{ISBN: 9780547928227, title: The Hobbit, author: J.R.R. Tolkien}],}
    - Third, we get the other results and add it to the response JSON
      - {present: [{ISBN: 0395520215, title: The Hobbit, author: J.R.R. Tolkien}],
         duplicate: [{ISBN: 9780547928227, title: The Hobbit, author: J.R.R. Tolkien}],
         result: [{ISBN: 1857939379, title: The Hobbit Companion, author: David Day}]}
    - Fourth, the frontend will put the three different types into there respective sections for the results.
