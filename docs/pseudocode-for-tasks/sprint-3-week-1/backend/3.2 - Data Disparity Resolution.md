# 3.2 - Data Disparity Resolution Between Local and OpenLibrary Data

### Main Objective:

Since we are taking user input data and data from OpenLibrary, there are certain situations that
can cause data anomalies. For this task we are negotiating between the two sources of data and
resolving the potential cases.


#### Task 3.2.1 - Case: ISBN already present in database

Note: Already has been addressed, noted for documentation purposes.


#### Task 3.2.2 - Case: Book not in OL database
Handled in: OL API 

Just return a response the OL does not have the book queried.


#### Task 3.2.5 - Case: Updating a Locally added book with OL data
Handled in: Mediator Module

Individual book entries will have an update button that can update a local book with OL data. We will make this
updating destructive for author, publisher, summary, and cover image.

- First, we call functions created for Task 3.2.4 to check and/or update authors/publishers with OLID.
- Second, we get the summary from OL and update the book record.
- Third, we get the cover image from OL and update the book record.
- Forth, we add the OLID to book entry.

With the addition of the OLID for the book entry, we can remove the update button and prevent a user from repeatedly updating.
